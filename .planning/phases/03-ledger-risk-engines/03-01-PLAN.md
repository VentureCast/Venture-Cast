<objective>
Build the PURE + PERSISTENCE double-entry ledger module for the AMM. This module turns
the pricing engine's fee output into a balanced set of double-entry postings and persists
them inside a caller-supplied Mongoose session.

Two clean layers, no orchestration:
- PURE (no DB): `buildBuyPostings` / `buildSellPostings` produce posting arrays; `assertBalanced`
  proves per-unit sum-to-zero BEFORE any write.
- PERSISTENCE (DB, session passed IN): `postEntries` inserts immutable LedgerEntry docs and
  $inc-upserts LedgerAccount projections. It does NOT start, commit, or abort the transaction —
  Phase 4 owns the transaction lifecycle.

Purpose: LEDG-01/02/03 — a trade posts balanced entries, projections are recoverable by summing
entries, and any unbalanced set is rejected before commit.
Output: services/amm/ledger/{postings,persist,index}.js + tests/unit/ledger.test.js
</objective>
