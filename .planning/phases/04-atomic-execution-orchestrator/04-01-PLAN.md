<objective>
Build the keystone money path: a single `executeOrder()` orchestrator that composes the
existing pricing, risk, and ledger engines behind ONE Mongoose transaction with optimistic
version locking, bounded retry, idempotency, quote expiry, and slippage enforcement.

This phase WIRES already-built pure modules (Phase 2 pricing, Phase 3 ledger + risk) and the
Phase 1 genesis transaction pattern. It does NOT add HTTP routes/controllers (Phase 5) and does
NOT touch the legacy `tradeService.js` path (flag-gated coexistence — the two engines live side
by side).

Purpose: every trade must settle atomically against the penny-balanced double-entry ledger with
a reserve that never goes negative, and concurrent/duplicate orders must never corrupt state.
Output: `services/amm/execution/` (orchestrator.js, priceTrade.js, index.js), an extended
`config.js` (per-tier risk caps), and a real-replset `tests/unit/execution.test.js`.
</objective>
